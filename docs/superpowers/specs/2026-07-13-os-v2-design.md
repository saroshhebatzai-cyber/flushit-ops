# Flush It OS v2 — Design Spec

**Date:** 2026-07-13
**Status:** User-approved (visual-companion session; mockups in `.superpowers/brainstorm/90495-1783915641/content/`)
**Predecessor:** 2026-07-12 Command Center (shipped, live at `7859965`)

## What this is

A visual and structural v2 of the Ops Hub shell: new light design language, a grouped
left sidebar replacing the ten top tabs, and a rebuilt Today surface. Everything else
is a reskin. Same single HTML file (`flushit-ops-hub.html` + identical `index.html`),
same Supabase data layer, same code-login, zero schema changes.

## Decisions made during exploration (do not re-litigate)

| Fork | Choice |
|---|---|
| Scope | Full v2: look + layout + flow ("D") |
| Architecture | Reskin the same machine — single file, vanilla JS ("A") |
| Visual direction | **Light Studio** — warm paper, white cards, green action color ("C") |
| Shell | **Grouped left sidebar** ("B") |
| Today | **Brief bar + focus stack merge** (approved as "Today v2") |

## 1 · Design language (token overhaul)

Replace the `:root` block. New tokens:

```css
--bg:#FAF8F4;          /* warm paper */
--surface:#FFFFFF;     /* cards */
--s2:#F3F0E9;          /* recessed fills (chips, inputs) */
--ink:#1A1A1A;         /* primary text */
--tm:#6E6A60;          /* secondary text */
--td:#B8B2A6;          /* muted/labels */
--border:#E9E4DA;      /* hairlines where shadows aren't enough */
--accent:#00B368;      /* green, action color on light surfaces */
--accent-bright:#00E87F; /* neon green — ONLY on dark panels */
--dark:#1A1A1A;        /* dark panel bg (brief bar, login card) */
--red:#E5484D; --amber:#F5A623; --cyan:#0090B3; --purple:#7C5CD6;
--shadow:0 1px 5px rgba(26,26,26,.07); /* single elevation level */
--r:12px; --rs:8px;
--display:'Syne',sans-serif;           /* big headers only */
--sans:-apple-system,'Inter',sans-serif; /* UI default */
--mono:'JetBrains Mono',monospace;     /* numbers, dates, statuses */
```

Rules:
- Body default becomes `--sans`; `--mono` is reserved for data (counts, dates,
  stage labels, badges) — this keeps the terminal DNA without the terminal gloom.
- `--accent-bright` never appears on light backgrounds (contrast); `--accent`
  never needs a glow. Remove all `--adim/--aglow`-style translucent glows.
- Red/amber only as 3px left-edge severity markers and small labels.
- Every existing color usage maps to the nearest new token during the reskin pass;
  status/stage color coding (per-editor colors, EDIT_TYPES colors) keeps hue meaning
  but re-tuned for light backgrounds (darker, fully saturated variants).

## 2 · Shell — grouped sidebar

Replace the top `<nav class="nav">` tab strip with a fixed left sidebar (~200px):

```
Flush It                ← wordmark, Syne
[＋ capture… (c)]       ← capture input, always visible (admin only)

MY DAY        (admin)   Today (badge: needsYou + inbox), My Week
DELIVERY                Projects, Queue (badge: my queue count), Calendar
GROWTH        (admin)   Leads, Research
STUDIO                  Team, Clients, Efficiency (admin)

Sarosh Ahmed · Out      ← footer: user + logout
```

- Active item: white pill + shadow; groups are small uppercase `--td` labels.
- `switchTab()` keeps its name/signature and admin guards; only the buttons move.
  `applyRoleGating()` hides whole groups instead of individual buttons where a
  group is fully admin-only.
- Header shrinks to: page title (current view name, Syne) + date + the six stat
  chips move INTO the Projects view header (they're pipeline stats, not app chrome).
- Responsive: below ~900px the sidebar becomes a slide-over behind a top-bar menu
  button; capture stays reachable in the top bar. (The app is desktop-first; the
  mobile layout must be usable, not perfect.)

## 3 · Today v2 (only structural rebuild)

Landing surface for admins, exactly per the approved mockup
(`today-merged.html`):

1. **Brief bar** — dark (`--dark`) rounded panel: date (Syne, white),
   `N NEED YOU` (`--accent-bright`, mono), `N INBOX · N IN PRODUCTION · N MEETINGS`
   (mono, gray), week focus right-aligned (read-only echo of My Week's focus).
2. **Needs You — ranked** — white cards, severity left-edge (red = blocking/review,
   amber = meeting today), each with a reason suffix ("blocking Tanzeel", "3:00 pm").
   Ranking: blockers (Sarosh Review videos, approval-stage projects) → today's lead
   meetings → everything else currently in needsYouHtml. Click-through behavior kept.
3. **Inbox — one-click sort** — existing inboxRowHtml semantics (Today/Tmrw/Mon
   chips, date picker, assign, file-under-research, done, delete) restyled: green
   filled chip on hover-primary, `--s2` pills otherwise.
4. **Team — live strip** — one pill per member: status dot (green = editing now,
   amber = has open loops, hollow = free) + name + one-line current item. Clicking
   a pill opens the existing person detail (current person-card content) as a
   drop-down panel under the strip. The two-column people-cards layout is retired.

## 4 · Reskin-only surfaces

Projects kanban, Queue, Calendar, My Week, Research, Leads, Team, Clients,
Efficiency, login screen: keep current DOM structure and behavior; restyle via
tokens + spacing pass (cards → white/shadow, fonts → sans, labels → mono).
Login: paper background, centered white card, one code field, dark button.
The WhatsApp ping panel and toasts get token colors, nothing else.

**Non-goals for v2:** no new features, no data-layer changes, no mobile app,
no per-surface layout rebuilds beyond Today, no dark-mode toggle (the brief
bar is the only dark element).

## 5 · Implementation slices

1. **Tokens + shell** — swap `:root`, global font/spacing pass, sidebar replaces
   tab strip, stat chips move into Projects header, responsive collapse.
   (Biggest visual diff, zero logic change.)
2. **Today v2** — rewrite `renderToday()` + its helpers (brief bar, ranked
   needs-you, team strip); inbox row restyle.
3. **Polish pass** — sweep every remaining surface against the tokens, kill
   stragglers (inline colors, glows), verify editor view, verify mobile collapse.

Each slice: implement → `node --check` + live browser verification (Playwright,
code login) → `cp` to `index.html` → commit → review → merge to main → push
(keychain token) → verify GitHub Pages.

## Success criteria

- Sarosh lands on Today and can answer "what now?" in under 10 seconds.
- Ten tabs become four labelled groups; nothing the team uses moves more than
  one click from where it was.
- Editors see zero admin UI (unchanged guarantee).
- No regression in any Task-10-verified behavior (sync, offline toast, gating,
  capture/sort, research filing).
